# 定时器服务 - 需求文档

## 1. 项目概述

### 1.1 项目背景

定时器服务是一个基于 Flask 的 Web 应用，提供定时器管理和实时事件推送功能。用户可以创建一次性定时器（once）或每日定时器（daily），当定时器触发时，系统会创建事件记录并通过 SSE（Server-Sent Events）实时推送给在线用户。

### 1.2 目标用户

- 需要定时提醒功能的个人用户
- 小型团队日常会议提醒

### 1.3 系统默认账号

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | admin | admin |
| 普通用户 | user | 123456 |

## 2. 功能需求

### 2.1 认证模块

#### 2.1.1 用户注册

- **功能描述**：新用户可以注册账号
- **输入要求**：
  - 用户名：3-50 个字符
  - 密码：至少 6 个字符
- **业务规则**：
  - 用户名不能重复
  - 密码使用 bcrypt 哈希存储
- **输出**：返回 user_id 和 username
- **错误处理**：
  - 400：输入验证失败
  - 409：用户名已存在

#### 2.1.2 用户登录

- **功能描述**：已注册用户登录系统
- **输入要求**：用户名 + 密码
- **业务规则**：
  - 验证用户名和密码
  - 登录成功后生成 JWT Token
- **输出**：返回 token、user_id 和 username
- **错误处理**：
  - 400：输入验证失败
  - 401：用户名或密码错误

#### 2.1.3 获取当前用户信息

- **功能描述**：通过 JWT Token 获取当前登录用户信息
- **输入要求**：Authorization 请求头
- **输出**：返回 user_id 和 username

### 2.2 定时器模块

#### 2.2.1 创建定时器

- **功能描述**：用户创建一个新的定时器
- **定时器类型**：

  | 类型 | 说明 | 参数 |
  |------|------|------|
  | once（一次性） | 单次提醒，触发一次后结束 | delay_seconds（1-86400秒） |
  | daily（每日） | 每天重复触发 | time_of_day（HH:MM:SS 格式） |

- **输入示例**：

  ```json
  // once 类型
  {
    "name": "提醒开会",
    "type": "once",
    "delay_seconds": 300
  }

  // daily 类型
  {
    "name": "晨会提醒",
    "type": "daily",
    "time_of_day": "08:00:00"
  }
  ```

- **业务规则**：
  - 定时器创建后自动启用
  - 不支持暂停功能
  - 名称不能为空
- **输出**：返回完整的定时器信息

#### 2.2.2 获取定时器列表

- **功能描述**：获取当前用户的所有定时器
- **查询参数**：
  - `status`：过滤状态（enabled/completed/deleted）
  - `type`：过滤类型（once/daily）
- **业务规则**：
  - 默认排除已删除的定时器
  - 按创建时间倒序排列

#### 2.2.3 获取单个定时器

- **功能描述**：根据 ID 获取定时器详情
- **错误处理**：定时器不存在返回 404

#### 2.2.4 更新定时器

- **功能描述**：修改定时器的参数
- **输入要求**：
  - once 类型：可修改 delay_seconds
  - daily 类型：可修改 time_of_day
- **业务规则**：
  - 已删除或已完成的定时器不可修改
  - 修改后自动重新计算触发时间

#### 2.2.5 删除定时器

- **功能描述**：删除定时器（软删除）
- **业务规则**：
  - 状态变为 deleted
  - 从调度器中移除
  - 数据不物理删除

### 2.3 事件模块

#### 2.3.1 获取事件列表

- **功能描述**：获取定时器触发事件的历史记录
- **查询参数**：
  - `unread_only`：0|1，仅返回未读事件
  - `limit`：返回数量限制（默认 100，最大 1000）
  - `cursor`：分页游标
- **输出字段**：
  - id、timer_id、timer_name
  - event_type（固定为 timer_fired）
  - is_read（是否已读）
  - fired_at、read_at

#### 2.3.2 标记事件为已读

- **功能描述**：将指定事件标记为已读
- **业务规则**：幂等操作，已读的事件重复调用返回成功

#### 2.3.3 批量标记事件为已读

- **功能描述**：将所有未读事件标记为已读
- **输出**：返回处理的事件数量

### 2.4 实时推送模块

#### 2.4.1 SSE 连接

- **功能描述**：建立持久连接，接收实时事件推送
- **连接方式**：
  - 请求头：`Authorization: Bearer <token>`
  - 或查询参数：`?token=<token>`
- **事件类型**：
  - `connected`：连接成功
  - `timer_fired`：定时器触发
  - `ping`：心跳保活（30 秒间隔）

## 3. 非功能需求

### 3.1 性能需求

- 支持多用户并发访问
- SSE 连接保活：30 秒心跳
- 列表查询默认限制 100 条，最大 1000 条

### 3.2 安全需求

- 密码使用 bcrypt 哈希存储
- JWT Token 认证，有效期 24 小时
- 用户数据隔离（只能访问自己的定时器和事件）
- SQL 注入防护（使用 ORM）

### 3.3 可用性

- 定时器创建后自动启用
- 定时器触发后自动推送通知
- 支持深色模式

## 4. 数据模型

### 4.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| username | VARCHAR(50) | 用户名（唯一） |
| password_hash | VARCHAR(255) | 密码哈希 |
| created_at | DATETIME | 创建时间 |

### 4.2 定时器表 (timers)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 所属用户 |
| name | VARCHAR(200) | 名称 |
| type | ENUM | once/daily |
| status | ENUM | enabled/completed/deleted |
| delay_seconds | INT | 延迟秒数（once） |
| time_of_day | TIME | 每日触发时间（daily） |
| fire_at | DATETIME | 计划触发时间（once） |
| next_fire_at | DATETIME | 下次触发时间 |
| last_fired_at | DATETIME | 上次触发时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 4.3 事件表 (timer_events)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 所属用户 |
| timer_id | INT | 关联定时器 |
| fired_at | DATETIME | 触发时间 |
| read_at | DATETIME | 已读时间 |

## 5. API 概览

| 模块 | 方法 | 路径 | 说明 |

| 事件 | POST | /api/events/ack_all | 全部标记已读 |
